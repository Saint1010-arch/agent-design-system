# 🦞 Agent Design System — 产品级 AI Agent 设计体系

> **不是又一个 SOUL.md 模板集合。是一套从需求到上架的完整 Agent 工厂。**

你见过很多 AI Agent 的 prompt 模板。但大多数只是"写一段人设描述"——没有质检标准、没有方法论来源追溯、没有解决 AI "模拟理解 ≠ 真正理解" 的工程手段。

这个项目提供的是：**一套经过体系化设计的 Agent 生产系统**，让你能批量生产**产品级、可上架、经过质检**的 AI Agent 角色配置。

---

## ✨ 这个项目能帮你做什么

- 🏗️ **用五层架构模板** 设计任何领域的专家级 Agent（Profile → Soul → Knowledge → Methodology → Protocol）
- 🎯 **用审美工程方法论** 解决 AI 最致命的问题——输出"看起来对但其实不对"的内容
- 🔍 **用质检清单** 在交付前发现 Agent 配置的漏洞
- 📐 **用生产手册 SOP** 把"模糊的需求"变成"完整的 Agent"
- 🧠 **用思维模型库** 给 Agent 注入真正的专家级思考方式

## 🔥 核心亮点：审美工程

大多数 Agent 设计只关心"让 AI 做什么"。我们额外解决了一个更深层的问题：

> **AI 模拟的"理解"和真正的"理解"之间有 gap，怎么用工程手段弥合？**

我们的回答是 8 种可操作的工程手段 + 3 种检测方法：

### 工程手段

| 手段 | 解决什么 |
|:---|:---|
| **定义锚定** | 防止"用对了词但理解偏了" |
| **行业顶级对标** | 把"好"从形容词变成可对照的具体标准 |
| **反例检验** | 防止确认偏误 |
| **来源追溯** | 区分"AI 生成的"和"有权威来源的" |
| **置信度标注** | 🟢确定 / 🟡推断 / 🔴推测——防止猜测当确定 |
| **能力圈声明** | 不懂就说不懂，不编 |
| **交叉验证** | 防止单一来源偏见 |
| **版本意识** | 防止用过时方法做当下的事 |

### 检测方法

- **删除测试**：删掉这句话，读者损失信息了吗？没有就删。
- **替换测试**：把术语换成其他领域的，还通顺吗？通顺说明是模板化空话。
- **追问测试**：追问"为什么"，只能答"通常来说"？深度不够。

### 虚假表现检测

来自 [Human 3.0](https://letters.thedankoe.com/p/human-30-a-map-to-reach-the-top-1) 的 False Transformation 思想——不只定义"什么是好的"，更精确定义"什么看起来好但其实不好"：

- ❌ **虚假的"用户思维"**：嘴上说"以用户为中心"，做决策时从未引用过一条用户数据
- ❌ **虚假的"战略思维"**：能说出 SWOT、波特五力，但无法判断当前场景该用哪个
- ❌ **虚假的"技术深度"**：堆砌术语但无法解释术语之间的因果关系

## 📖 如何使用

### 场景一：从零设计一个 AI Agent

> 适用于：你有一个模糊的需求（比如"我想要一个产品经理 Agent"），想把它变成一份完整的、AI 读了就能运行的配置。

**Step 1：明确需求**

先回答 4 个问题（参考 [`templates/生产手册.md`](templates/生产手册.md) 的 Phase 1）：

| 问题 | 你的回答 |
|:---|:---|
| 这个 Agent 做什么？（核心职责，不超过 3 个） | 例：帮我做竞品分析并出对比报告 |
| 服务谁？ | 例：产品团队 |
| 什么说话风格？ | 例：严谨专业，但不要太闷 |
| 现实中最像谁？ | 例：麦肯锡的资深咨询师 |

**Step 2：填写五层架构模板**

打开 [`templates/五层架构模板.md`](templates/五层架构模板.md)，逐层填写：

```
第一层 Profile    → 名字、MBTI、Slogan、行业对标人物
第二层 Soul       → 灵魂准则、审美标准、行为禁令
第三层 Knowledge  → 概念定义表（含"虚假表现"列）、行业对标、跨领域因果链
第四层 Methodology→ 方法论（必须标注来源 + 适用场景 + 典型陷阱）
第五层 Protocol   → 沟通风格、回复示例（至少2段）、行为边界三栏表
```

> 💡 **关键提醒**：第三层的"概念定义表"是整个体系的灵魂。不要跳过"虚假表现"列——它能防止你的 Agent 输出"看起来专业但实际空洞"的内容。

**Step 3：质检**

用 [`templates/质检清单.md`](templates/质检清单.md) 逐项打勾：

- 🟢 全部通过 → 可以交付使用
- 🟡 1-2 项未通过 → 标注问题，修复后交付
- 🔴 3+ 项未通过 → 打回重做

**Step 4：投入使用**

把填好的配置文件放进你的 AI Agent 平台（OpenClaw、ChatGPT Custom Instructions、Claude Projects 等）。配置即灵魂——AI 读了就能按照你设计的角色工作。

---

### 场景二：改进一个已有的 Agent

> 适用于：你已经有一个 Agent 在用，但总觉得它"不够好"——回答太泛、不像专家、容易说空话。

**Step 1：用质检清单诊断**

拿 [`templates/质检清单.md`](templates/质检清单.md) 对照你现有的 Agent 配置，看哪些项不通过。最常见的问题：

| 症状 | 根因 | 解法 |
|:---|:---|:---|
| 回答太泛，像实习生 | 缺少行业顶级对标 | 补第三层的行业对标（具体到人/公司/做法） |
| 说话正确但没有信息量 | 没有反压缩规则 | 在行为禁令里加"不要为了精简而压缩回复" |
| 看起来专业但经不起追问 | 缺少概念定义表 | 补第三层的关键概念定义 + 虚假表现列 |
| 所有话题都答，但都不深 | 没有能力圈边界 | 在灵魂准则里加自我校准机制 |

**Step 2：针对性补强**

按诊断结果，回到 [`templates/五层架构模板.md`](templates/五层架构模板.md) 中对应的层级补充内容。

### 场景三：管理多个 Agent 团队

> 适用于：你需要多个 Agent 协作（如产品经理 + 技术调研 + 文案），要避免角色重叠和职责混乱。

用 [`templates/角色矩阵.md`](templates/角色矩阵.md) 管理全局：
- 记录每个 Agent 的核心能力域
- 确保能力覆盖不重叠
- 画出协作拓扑（谁的输出是谁的输入）

## 📐 设计原则

### Prompt 工程十诫

1. 角色 > 指令
2. 禁止清单 > 鼓励清单
3. 示例 > 描述
4. 结构 > 自由发挥
5. 理由 > 命令
6. 具体 > 泛泛
7. 分层 > 堆砌
8. 最小权限
9. 可测试
10. 人格是杠杆

### 设计黄金三角

```
        [角色认同]
         /       \
  [方法论注入] — [行为边界]
```

三条边上都站着**审美**——角色认同要有审美（不是空壳人设），方法论要有审美（不是空洞框架），边界要有审美（不是一刀切的粗暴限制）。
